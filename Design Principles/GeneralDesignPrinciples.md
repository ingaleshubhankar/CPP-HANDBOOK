# [General Design Principles](#general-design-principles)

### [DRY (Don’t Repeat Yourself)](#dry-dont-repeat-yourself)

**Definition:** 
Avoid duplicating logic—keep a single source of truth.

**Explanation:**
Repeated logic increases maintenance cost and risk of inconsistency.

**Why it matters:**
 - Easier updates
 - Cleaner codebase

**Example:**<br>
Move repeated logic into reusable functions.
<br>
<br>

### [KISS (Keep It Simple, Stupid)](#kiss-keep-it-simple-stupid)

**Definition:**
Prefer simple solutions over complex ones.

**Explanation:**
Avoid unnecessary complexity. Simpler code is easier to read, debug, and maintain.

**Why it matters:**
 - Reduces bugs
 - Improves clarity

**Example:**
Avoid over-engineering for simple problems.
